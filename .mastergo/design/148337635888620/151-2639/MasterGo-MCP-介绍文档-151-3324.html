MasterGo MCPEditorial Architecture Manual
MasterGo MCP / 2.0
MasterGo MCP赋予 AI 掌控画布的原生能力

这不是一项附加功能。而是一份智能体与设计工具之间的双向契约。

Phase 1 / Initialization引擎介入，三步配置
01
新增 MCP 服务配置

在 Cursor, VSCode 或 Trae 等现代开发环境中，打开系统配置并介入 mastergo 服务节点。或者在 AI 对话框让它自己参考右边的代码为你自动配置。

02
保持本地服务在线

无需填写 access key，IDE 会连接本机 MasterGo MCP 服务。

03
重启并保持在线

等待引擎灯亮起。MasterGo MCP 在线后，AI 的所有调用才会穿透层级，直达画布。

mcp_config.json

{
    "mcpServers": {
        "mastergo": {
          "command":"npx",
          "args": [
              "-y",
              "@mastergo/vibe-mcp",
              "--url=http://localhost:50678"
          ]
       }
    }
}

Phase 2 / Capabilities全景能力矩阵
Design + Create

打破边界的生成能力。从一句话需求直接跃迁至完整的画布设计稿，提供最短路径的产品形态验证。

生态组件映射

严格遵循企业级设计系统。支持跨团队库挂载与智能组装生成策略。

非破坏性读取

深度解析当前选中上下文与视觉层级，确保在零修改风险下获取全量代码结构。

微观局部重构

像手术刀一样精准。利用自然语言下发指令，瞬间完成色彩、文案的局部调整。

原子节点置换

打破繁琐的操作流。一键识别并替换图标库、图片占位符或其他逻辑节点。

图层剥离

静默且安全地删除单一或多选的冗余图层，保持画布极简。

多维差异对比

构建视觉高亮。实时比对线上画布状态与本地构建基准之间的差异。

代码逆向覆盖

形成完美闭环。将全量代码结构，直接投射覆盖至设计稿。

资产快照流转

利用唯一 Id，在研发团队内部无缝提取并分享设计资产片段。
