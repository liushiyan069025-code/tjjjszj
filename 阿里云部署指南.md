# 减脂打卡 - 阿里云部署指南

## 架构说明

```
用户浏览器
    │
    ▼
阿里云 ECS (公网 IP)
    │
    ├── Nginx (80/443) ── 反向代理 + HTTPS
    │       │
    │       ▼
    └── Node.js server.js (3000)
            │
            ├── 静态文件托管 (dist/)
            └── AI 代理 (/api/ai-proxy)
                    │
                    ▼
                第三方 AI API (OpenAI / Anthropic / 等)
```

**核心要点**：AI 代理中间件从 Vite dev server 迁移到了独立 Node.js 服务器（[`server.js`](server.js:1)），解决了纯静态部署导致 AI 功能失效的问题。

---

## 前置条件

1. **阿里云 ECS 实例**：已购买，可访问公网
2. **操作系统**：Ubuntu 22.04 / CentOS 7+ / Alibaba Cloud Linux 3
3. **安全组**：已开放 80（HTTP）、443（HTTPS）、22（SSH）端口
4. **域名**（可选）：已解析到 ECS 公网 IP

## 部署步骤

### 第一步：服务器环境准备

SSH 登录到 ECS 后，安装 Node.js 和 PM2：

```bash
# 安装 Node.js 20.x LTS（使用 nvm）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
nvm use 20
node -v   # 确认 v20.x

# 安装 PM2（进程守护，崩溃自动重启）
npm install -g pm2

# 安装 Nginx
sudo apt update && sudo apt install -y nginx   # Ubuntu
# 或
sudo yum install -y nginx                       # CentOS
```

### 第二步：上传项目代码

**方式 A：Git 克隆（推荐）**

```bash
cd /opt
git clone <你的仓库地址> jianzhi-daka
cd jianzhi-daka
```

**方式 B：本地打包上传**

在本地执行：
```bash
cd 减脂打卡
npm run build                    # 构建前端到 dist/
# 打包必要文件（排除 node_modules）
tar czf deploy.tar.gz \
  dist/ server.js package.json ecosystem.config.cjs public/
# 上传到服务器
scp deploy.tar.gz root@<ECS公网IP>:/opt/
```

在服务器上：
```bash
cd /opt
mkdir -p jianzhi-daka && tar xzf deploy.tar.gz -C jianzhi-daka
cd jianzhi-daka
```

### 第三步：安装依赖并启动

```bash
cd /opt/jianzhi-daka

# 安装生产依赖
npm install --production

# 用 PM2 启动（进程守护 + 开机自启）
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup    # 按提示执行返回的命令，实现开机自启

# 确认运行状态
pm2 status
pm2 logs jianzhi-daka
```

### 第四步：配置 Nginx 反向代理

```bash
sudo nano /etc/nginx/conf.d/jianzhi-daka.conf
```

写入以下配置（将 `your-domain.com` 替换为你的域名或 IP）：

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 替换为你的域名或 ECS 公网 IP

    # 静态文件 + SPA fallback
    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # AI 代理 —— 关键：支持 SSE 流式响应
    location /api/ai-proxy {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # SSE 流式支持
        proxy_buffering off;           # 关闭缓冲，实时转发
        proxy_cache off;
        proxy_set_header Connection '';  # 清除 Connection 头
        chunked_transfer_encoding on;

        # 超时设置（AI 响应可能较慢）
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|svg|ico|woff2?)$ {
        proxy_pass http://127.0.0.1:3000;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

测试并重载 Nginx：
```bash
sudo nginx -t          # 检查配置语法
sudo systemctl reload nginx
```

### 第五步（可选）：配置 HTTPS

使用 Let's Encrypt 免费证书：

```bash
sudo apt install -y certbot python3-certbot-nginx   # Ubuntu
sudo certbot --nginx -d your-domain.com
# 按提示完成，certbot 会自动修改 Nginx 配置并设置定时续期
```

## 常见问题

### Q: AI 对话功能不工作？
检查 Nginx 的 `/api/ai-proxy` 配置中 `proxy_buffering off;` 是否生效。流式响应（SSE）必须关闭缓冲，否则响应会被延迟。

### Q: 页面白屏？
确认 `dist/` 目录已正确构建并上传。运行 `npm run build` 后检查 `dist/index.html` 是否存在。

### Q: 端口被占用？
修改 `ecosystem.config.cjs` 中的 `PORT` 值，同时更新 Nginx 配置中的 `proxy_pass`。

### Q: 如何不使用 Nginx 直接暴露？
可以省略 Nginx，直接开放 3000 端口访问。但建议使用 Nginx 以获得 HTTPS、缓存、日志等能力。
